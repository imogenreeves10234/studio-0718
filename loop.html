No mystery here

01What an AI "loop" really is

A loop is an LLM + some tools, run on a schedule against a goal, that generates something → does something with it → checks how it went → adjusts → repeats. That's Anthropic's own definition of an agent: "LLMs using tools based on environmental feedback in a loop." The tweet's cycle — make content → post → read engagement → change next batch — is a completely standard shape. "Midjourney locks the face, Claude writes, a scheduler posts" is just any image model + any LLM + any cron job. The paid guide sells the wiring diagram, and the wiring diagram is free.

The honest core: loop engineering is an operations skill — state, observability, blast radius, the discipline to let a thing fail cheaply and restart clean. The harness (scripts/scheduler) is the fragile part, not the AI. And for your model specifically: 2026 FB Content-Monetization RPMs are pennies and the algorithm actively suppresses faceless AI slop while favoring distinct personalities — so a consistent, characterful, disclosed persona is your edge. Build the loop because it makes you fast and consistent, not because it prints money while you sleep.

Load-bearing

02The 8 principles that make a loop reliable

Synthesized from Anthropic + people who run loops in production. Ignore these and a loop is "a very expensive hallucination."

  1. Define "done" precisely and cap iterations. "If you can't say what 'done' looks like, you don't have a loop — you have a wish." Most real loops need 3–5 iterations; more means it's stuck.
  2. A governing constitution. One always-loaded rulebook (CLAUDE.md) encoding voice, hard rules, success criteria — so behavior is consistent across every run.
  3. Real feedback before acting (evals, not vibes). A separate CHECKER sub-agent reviews the primary agent's work. This is where your judgment lives.
  4. Guardrails + a kill-switch. Hard guardrails are code that physically blocks an action (a publish script that refuses without an approval flag). Start read-only; grant post permission last.
  5. Human-in-the-loop checkpoints. The agent pauses for sign-off at defined gates and whenever it hits a blocker.
  6. Persistent state on disk, not in the context window. Every concept, score, decision written to a file/git before the next step — a truncated conversation never erases memory.
  7. Observability + graceful failure + small blast radius. Log everything; let it fail cheaply and restart clean. "Build the part that pages you before the part that impresses you."
  8. Start simple. A cron + a good prompt gets ~80% of "adaptive" behavior. Add agentic complexity only when a plain prompt + a metrics file falls short.
The metric-gaming trap: every loop optimizes the number you gave it, not the thing you meant. A caption loop told to maximize "engagement" will happily drift toward clickbait and rule-breaking. Make your KPI retention + per-view earnings, not likes.

The rulebook

03The CLAUDE.md "constitution" pattern

A CLAUDE.md is a persistent rulebook the agent reads automatically at the start of every session — including every scheduled cloud run — before any prompt. You never invoke it; it's just always there. It's not a style guide; it's where non-negotiable rules + success criteria live so behavior stays consistent. (Karpathy's 4-rule CLAUDE.md reportedly cut agent errors ~41% → ~11%.) Load order: ~/.claude/CLAUDE.md → ./CLAUDE.md → ./CLAUDE.local.md.

📜 CLAUDE.md — persona constitution (fill the <...> slots)
# PERSONA CONSTITUTION — <Persona Name>  (agent: read this before every action)

## 0. What I am
- <Name> is an OPENLY AI-GENERATED persona. One-person operation.
- I (the agent) draft concepts, prompts, and captions. I NEVER publish.
  A human approves every batch via `batches/<date>/approved.flag`.
  No flag = I stop.

## 1. Disclosure (non-negotiable — legal + platform survival)
- Bio states "AI-generated / virtual creator." Every caption includes `#AI`
  + a plain-language line (e.g. "AI-generated character").
- Keep C2PA / AI metadata on all media INTACT. Never strip it to dodge labels.
- NEVER claim or imply <Name> is a real human — in captions, comments, or DMs.
- Undisclosed AI is an active strike category. Disclosure protects the whole
  account and monetization eligibility. Treat it as a kill-switch condition.

## 2. Voice
- Tone: <warm / dry-funny / hype — pick ONE and hold it>.
- Lines: 3-8 words on-screen; hook in the FIRST line; payoff by the end.
- Yes: <topics/vibes>.  No emoji walls. No hashtag spam (max <n>).
- Full samples + banned phrases in `constitution/voice.md`.

## 3. Visual lock
- Use `constitution/visual-lock.md` identity block on EVERY still
  (SPECIES/FACE/HAIR/WARDROBE/DO-NOT). Run drift check vs reference
  embedding; flag anything over threshold for human review.

## 4. DO
- SFW, positive, broadly relatable, trend-aware but on-brand.
- Propose 3-5 clean concepts/day. If fewer than 3 survive the rules,
  STOP and ask — do not fill the batch with junk.
- Optimize for RETENTION and per-view earnings, not likes.

## 5. DO-NOT (hard reject — these void `constitution_check`)
- NSFW / suggestive / anything sexualizing the persona.
- Minors, or content aimed at minors.
- Health, medical, weight-loss, financial, or earnings claims/advice.
- Politics, religion, protected-group commentary, real-person likeness/deepfake.
- "DM me for...", giveaways, off-platform funnels, engagement-bait spam.
- Auto-DMing strangers. DMs / comment replies are HUMAN-handled.
- Anything requiring me to imply I am human.

## 6. Posting rules
- Publish ONLY via the official Meta path (Business Suite Planner, or Pages
  Graph API `scheduled_publish_time`). NEVER browser-automation posters.
  Pages only, never groups.
- Max <N> posts/day, hard cap. Never publish without approved.flag.

## 7. KPIs (priority order)
1. 3-second / average watch retention
2. Per-view earnings + RPM (Content Monetization dashboard)
3. Follower growth toward monetization thresholds
4. Save / share rate
- Likes/comments are weak signals, not targets.

## 8. Kill-switch (HALT loop, notify human, do NOT self-resume) if ANY:
- A policy strike, monetization warning, or reach/earnings crash overnight.
- Face-drift flagged on >1 clip in a batch.
- Fewer than 3 constitution-clean concepts available.
- A trend seed touches any DO-NOT topic.
- Publish auth fails, or metrics come back all-zero (tracking break).
- Human places a `HOLD` file in the repo root.

Supporting files: constitution/voice.md (voice samples + a banned-phrase list your caption linter checks) and constitution/visual-lock.md (the identity prompt block used on every still — your VegToons character-lock format).

You already own it

04How to build it — Claude Code does this natively

The tweet's "constitution + scheduler + integrations + autonomous execution" isn't a product you buy. It maps 1:1 onto features you already have:

  • CLAUDE.md = the persistent constitution (what to do).
  • A Routine on a cron schedule (/schedule) = the automated trigger (when), running on Anthropic's cloud so your laptop can be closed.
  • MCP connectors = the integrations (a social publisher, etc.).
  • Hooks in settings.json = deterministic safety rules that always fire.

The native toolset

  • /loop — runs a prompt on an interval inside an open session. Good for polling while you work; not for unattended automation (dies when you close the terminal, auto-expires after 7 days).
  • /schedule → Routines — the real automation layer. Persistent cloud agents on cron (0 9 * * * = daily 9am), or triggered by an API POST / GitHub event. Each run gets a full session, clones your repo fresh (so it loads ./CLAUDE.md), runs autonomously, exits. Min interval 1 hour. Test with "Run now" first.
  • Hooks — shell commands at lifecycle events (PreToolUse to block unsafe commands, Stop to evaluate "is the work done?").
Sandbox caveat: cloud Routines run in a sandbox — they can't touch your local files or local Veo rig, only the cloned git repo + declared env vars + declared MCP connectors. That's why the design below keeps generation local and uses the repo + an MCP publisher as the bridge.

🔧 loop.sh — one cycle (drafts only, never publishes)
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"
DATE=$(date +%F)

# Global kill-switch
[ -f HOLD ] && { echo "HOLD present — halting."; exit 0; }

# 5) READ yesterday's numbers (read-only, safe)
python3 scripts/pull_insights.py --out "data/metrics/${DATE}.json" || \
  { echo "Insights pull failed — halting, not retrying."; exit 1; }

# 1+3+6) IDEATE, WRITE, ADAPT — drafts only, never publishes
claude -p "Read CLAUDE.md. Run today's cycle for ${DATE}:
  1. Ideate 3-5 constitution-clean concepts -> batches/${DATE}/concepts.json
     (each with constitution_check). If fewer than 3 survive, STOP and write
     batches/${DATE}/NEEDS_HUMAN.md instead.
  3. Draft captions.md with the disclosure line; pass the banned-phrase linter.
  6. Update playbook.md from the last 14 days of metrics; write tomorrow's brief.
  Do NOT publish. Do NOT create approved.flag."

# 2) GENERATE runs on the local rig (prompts written above); face-drift QA is human.
# 4) PUBLISH is intentionally NOT here — a human reviews the batch, drops
#    batches/${DATE}/approved.flag, then runs: python3 scripts/publish.py --date ${DATE}

echo "Cycle drafted for ${DATE}. Awaiting human approval before any publish."

Earn the autonomy

06Crawl → Walk → Run

Automate the plumbing first, the judgment last. Trust is earned by clean cycles, not by a calendar.

🐣 Crawl (weeks 1–3) — prove quality & safety

Agent proposes concepts, prompts, captions. Human does everything else — QA every clip, paste-post manually via Business Suite, review Adapt daily. Exit when: ~2 weeks of on-voice, on-model, zero-flag batches with disclosure correct every time.

🚶 Walk (weeks 3–6) — automate plumbing, keep ONE daily gate

Auto: insights pull, ideation, generation, caption drafting, and scheduling via the Graph API — but only after approval. Human: one ~10–15 min daily approve gate (review batch + captions, drop approved.flag) + the face-drift contact sheet. Exit when: ~2–3 weeks where your daily edits trend to zero.

🏃 Run (week 6+) — unattended most days, exception-gated

Full cycle runs unattended on normal days. Realistic Run ≠ "6 weeks untouched" — it's daily auto-run + exception-based human intervention + a weekly strategy review. Auto-HOLD triggers (reach/earnings drop, any strike, drift flag, DO-NOT trend, auth/metric failure) halt the loop and ping you. You never fully walk away — the failure mode (a strike, or a drifted run posting for days) is expensive and slow to undo.

Straight talk

07Honest reality & guardrails

Disclosure is not optional — it's survival. Meta made AI labeling mandatory in 2026 and auto-detects AI media via C2PA metadata (which Veo output carries). So: bio says "AI-generated," every caption carries the disclosure, metadata stays intact (stripping it to dodge the label is the fast track to a strike), and nothing ever implies she's a real human. One strike endangers the whole account and your Content-Monetization eligibility.

Automate at your peril (keep human)

  • The publish decision — the non-negotiable gate.
  • Face-drift QA — a wrong-face clip is brand poison.
  • Disclosure + policy compliance.
  • DMs & comment replies — auto-DM = spam/inauthentic + deception risk.
  • New/edgy trend seeds; anything involving spend or off-platform funnels.

Safe to run unattended

  • Insights pull + aggregation.
  • Ideation / prompt / caption / playbook drafts.
  • Scheduling API calls for content that already passed the gate.
  • Bookkeeping (post-ID logging, dedupe, git commits).
What breaks: the harness most of all (scripts/scheduler = the part most likely to break for no reason — build alerting before cleverness); the eval via metric-gaming (keep KPIs on retention + per-view earnings); and the economics (2026 FB RPMs are pennies, faceless AI is suppressed — a distinct, disclosed persona is your only real edge). The loop makes you fast and consistent; don't let the tooling convince you the money is automatic.
