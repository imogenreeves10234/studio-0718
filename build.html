Do first

0Prep the machineonce

The #1 thing that breaks new installs: your 5090 and PRO 6000 are Blackwell cards. Normal PyTorch won't run on them — ComfyUI will error or fall back to CPU. You MUST use the CUDA 12.8 build (step below). Miss this and nothing works.
  1. Update NVIDIA driver to 570+ and install CUDA Toolkit 12.8+.Required for Blackwell (sm_120). Older CUDA = dead on arrival.
  2. Install Python 3.11 (or 3.12), git, and ffmpeg (ffmpeg must be on PATH).Avoid Python 3.13 — some nodes lag. ffmpeg handles all video in/out.
  3. Make a free HuggingFace account (you'll accept the Flux license to download it).
  4. Free ~500 GB on a fast NVMe SSD for models.Slow HDD = everything crawls. This is not optional at this scale.

Make the girl

2Create the personaonce per girl

  1. Hero face. Run the corrected seed prompt in ComfyUI (Flux.1 dev), make ~10, pick the single best face. Save it.Young + attractive is the whole hook — use the fixed prompt, not the old one.
  2. Dataset. From that hero, generate 20–30 varied shots (different angles/expressions) with PuLID keeping the face identical. Keep the ~25 cleanest.
  3. Train her LoRA. Feed those ~25 into FluxGym, ~1,500–2,000 steps → one file that IS her face forever.
  4. Build the Identity Bank. Generate her locked front / 3-4 / profile / medium / full-body stills (LoRA + PuLID + ControlNet + FaceDetailer → SUPIR). You reuse these as reference on every future image.

The repeatable part

3Make each videorepeat

  1. Still. Generate the shot in ComfyUI (LoRA + PuLID + ControlNet pose + FaceDetailer), then finish with SUPIR + a touch of grain.
  2. Animate the still → pick one:Free: Wan 2.2 (local, run it on your 2nd GPU).  •  Best: Veo 3 via your veo-flow-multi — for money shots + when you want built-in audio. Use the 8 Veo prompt templates.
  3. Only if she talks: clone her voice with Chatterbox (free, commercial-safe), then lip-sync with LatentSync.Skip this entirely for silent + voiceover-in-edit clips.
  4. Edit. Assemble 9:16 in CapCut (easiest auto-captions) or Kdenlive (fully free). Add the hook text, captions, CTA; cover any watermark.
  5. Post manually to Facebook.

The one choice you asked about

Free (Wan 2.2) vs Veo 3

Wan 2.2 — free, local

  • All your volume, drafts, and iteration — zero cost.
  • Runs on your beast; commercial use allowed.
  • Silent — you add voice/music in the edit.

Veo 3 — paid, for money shots

  • Higher cinematic fidelity + better physics on hard motion.
  • Built-in synced audio/dialogue in the clip.
  • Use only for the few hero clips that need the polish.

Simple rule: Wan for the daily batch, Veo 3 for the occasional showpiece. You can ship the whole thing with $0 of video if you want.

Is this everything? The MUST-haves

✅ Yes — the core kit (all free/local except Veo)

ComfyUI · Flux.1 dev · PuLID · ControlNet Union-Pro · FaceDetailer · FluxGym (LoRA) · Wan 2.2 (free video) — plus Veo 3 only for money shots. That's the pipeline, start to finish.

Don't-forget MUSTs (these silently break it if missed):
  • Blackwell PyTorch: the cu128 build + driver 570+ / CUDA 12.8+, and no xformers — or ComfyUI won't run on your cards.
  • FaceDetailer needs its detector model (face_yolov8m.pt) — it does nothing without it.
  • ffmpeg on PATH · HuggingFace login (Flux is gated) · onnxruntime-gpu · ~500 GB fast SSD.
  • Two GPUs = two ComfyUI instances (--cuda-device 0 / --cuda-device 1): run images on one card, Wan video on the other. There's no auto-pooling.
Only if the persona talks: add Chatterbox (voice clone) + LatentSync (lip-sync). For silent/voiceover clips you don't need either.

That's the whole kit — nothing else is a MUST. Everything beyond this list (extra models, fancier upscalers, FLUX.2 for hero stills) is a nice-to-have, not required to ship.
